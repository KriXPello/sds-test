# Тестовое задание для компании "Смарт Дельта Системс".

Краткие требования:

- Функциональные
  1. Автоматическое определение местоположения при загрузке приложения
  2. Отображение текущей погоды: температура, иконка, описание, влажность, скорость ветра
  3. Поиск города вручную через поле ввода
  4. Обработка ошибок: геолокация, API, сетевые ошибки
  5. Индикатор загрузки при запросах

- Технические
  - Сборка проекта через Webpack
  - API ключ вынесен в .env файл
  - Адаптивная верстка (мобильные устройства и десктоп)
  - Рабочая продакшн-сборка через npm run build


Полные требования к заданию: [файл задания](./задание.pdf)

# Инструкция по установке и запуску

## Подготовка
В папке с проектом:

- Установить зависимости командой `npm i` или `npm ci`
- Создать файл `.env` и скопировать в него содержимое `.env.example`, заполнить необходимые переменные


## Запуск в режиме разработки
- При необходимости создать файл `.env.development`
  - работает только при `mode=development`
  - при совпадении имен переменные из `.env` будут переопределены указанными
- выполнить `npm run dev`


## Сборка
- При необходимости создать файл `.env.production`
  - работает только при `mode=production`
  - при совпадении имен переменные из `.env` будут переопределены указанными
- выполнить `npm run build` (результат будет в dist/)



# Примечания к решению

## Локализация
openweathermap Geocoding API ограничен в плане локализации городов и областей, поэтому при поиске на русском в результатах можно увидеть названия на английском. Также возможно дублирование городов.

## Поиск городов
openweathermap API при поиске городов по названию часто находит их только по полному названию.

## Использование токена API
Сборка приложения для GitHub Pages будет запрашивать токен доступа к openweathermap API.
Токен сохраняется в sessionStorage, поэтому для сброса нужно закрыть страницу и открыть заново.

Важно! Если вы только создали токен, может быть ситуация, когда он еще не был активирован. Цитата из FAQ openweathermap о возможных причинах ошибки 401:

> Your API key is not activated yet. Within the next couple of hours, it will be activated and ready to use.

## Определение местоположения
При открытии приложения будет выполнена попытка определить город пользователя, используя данные геолокации.
- в случае успеха будет выведен диалог с информацией о городе, для подтверждения выбора.
- если не удалось определить местоположение будет выведено исчезающее сообщение об эттом.
- если пользователь уже выбирал город (предложенный на основе местоположения или найденный вручную), то автоматическое определение местоположения **не запустится**, но его можно запустить вручную на панели поиска города.
